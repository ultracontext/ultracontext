The UltraContext CLI watches agent session files in the background, parses events, and syncs them to the Context API in real time.

<Info>New here? Start with the [Quickstart](/quickstart) for a 2-minute setup.</Info>

## Supported agents

| Agent | Session files |
|-------|-------------|
| **Claude Code** | `~/.claude/projects/**/*.jsonl` |
| **Codex** | `~/.codex/sessions/**/*.jsonl` |
| **OpenClaw** | `~/.openclaw/agents/*/sessions/**/*.jsonl` |

## Context metadata

The daemon enriches each context with metadata you can use to filter and search:

| Field | Description | Example |
|-------|-------------|---------|
| `source` | Agent that created the session | `claude`, `codex`, `openclaw` |
| `user_id` | User identifier (`DAEMON_USER_ID` or `$USER`) | `alice` |
| `host` | Machine hostname | `Alices-MacBook-Pro.local` |
| `project_path` | Project directory (Claude Code only) | `/Users/alice/Code/myapp` |
| `session_id` | Original agent session ID | `b1ae8187-56cb-...` |
| `started_at` | Timestamp of the first event | `2026-03-08T00:01:20Z` |

These fields are filterable via `GET /contexts` and the [MCP Server](/fundamentals/mcp-server).

## CLI commands

```bash bash
ultracontext              # start sync (daemon + dashboard)
ultracontext sync         # start sync (daemon + dashboard)
ultracontext stop         # stop daemon
ultracontext config       # run setup wizard
ultracontext update       # update CLI globally
ultracontext version      # print version
ultracontext help         # show help
```

<Info>See [Configuration](/fundamentals/configuration) for config file, environment variables, and setup options.</Info>

## Updating

```bash bash
ultracontext update
```

Updates to the latest version. The package manager is auto-detected from your install.
